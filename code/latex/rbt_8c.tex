\doxysubsection{Detailed Description}
Implementation of a Red-\/\+Black \doxylink{structTree}{Tree} in C (Insertion Only). 

This file contains the definition and implementation details of a Red-\/\+Black \doxylink{structTree}{Tree}, a self-\/balancing binary search tree. In a Red-\/\+Black \doxylink{structTree}{Tree}, each node contains an extra bit for denoting the color of the node, either RED or BLACK. A Red-\/\+Black tree satisfies the following properties\+:


\begin{DoxyEnumerate}
\item Every node is either RED or BLACK.
\item Red nodes cannot have RED children; i.\+e. RED nodes can only have BLACK children.
\item NULL nodes are BLACK.
\item The root is always BLACK.
\item Every path from a node to its descendant NULL nodes has the same number of BLACK nodes.
\end{DoxyEnumerate}

The tree structure supports operations such as insertion, deletion, and search with guaranteed log(n) maximum time complexity for each operation, making it efficient for data retrieval and manipulation. Space complexity is linear; i.\+e. O(n).

Key Components\+:
\begin{DoxyItemize}
\item Color\+: An enumeration defining the color (RED, BLACK) of nodes in the tree.
\item \doxylink{structNode}{Node}\+: A structure representing a node in the Red-\/\+Black \doxylink{structTree}{Tree}, containing data, color, and pointers to its children and parent.
\item \doxylink{structTree}{Tree}\+: A structure representing the entire Red-\/\+Black \doxylink{structTree}{Tree}, encapsulating its root and size.
\end{DoxyItemize}
