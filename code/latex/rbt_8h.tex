\doxysubsection{Detailed Description}
Declaration of Red-\/\+Black \doxylink{structTree}{Tree} data structures and functions. 

This header file contains the declarations for the Red-\/\+Black \doxylink{structTree}{Tree} data structures and outlines the interface for operations on a Red-\/\+Black \doxylink{structTree}{Tree}. A Red-\/\+Black \doxylink{structTree}{Tree} is a kind of self-\/balancing binary search tree where each node stores an extra bit representing its color (RED or BLACK) to ensure the tree remains approximately balanced. This property enables the tree to support insertion, deletion, and search operations in O(log n) time, making it efficient for various applications. A Red-\/\+Black tree satisfies the following properties\+:


\begin{DoxyEnumerate}
\item Every node is either RED or BLACK.
\item Red nodes cannot have RED children; i.\+e. RED nodes can only have BLACK children.
\item NULL nodes are BLACK.
\item The root is always BLACK.
\item Every path from a node to its descendant NULL nodes has the same number of BLACK nodes.
\end{DoxyEnumerate}

Key Components\+:
\begin{DoxyItemize}
\item Color\+: An enumeration that defines the possible colors (RED, BLACK) of nodes in the tree.
\item \doxylink{structNode}{Node}\+: A structure representing a node within the Red-\/\+Black \doxylink{structTree}{Tree}. It includes the node\textquotesingle{}s data, its color, and pointers to its children and parent.
\item \doxylink{structTree}{Tree}\+: A structure representing the Red-\/\+Black \doxylink{structTree}{Tree} itself, encapsulating a pointer to its root and the total number of nodes in the tree.
\end{DoxyItemize}
