\doxysubsection{Detailed Description}
This section describes the helper functions used for maintaining the Red-\/\+Black tree properties. These functions are essential for internal operations such as rotations, insertions, deletions, and color adjustments which ensure the tree remains balanced and conforms to red-\/black tree rules. Key operations include\+:


\begin{DoxyItemize}
\item {\ttfamily \doxylink{group__rbt__helpers_ga4204e86d4f09d468b86d4e333518b705}{grandparent()}}\+: Returns the grandparent of a given node.
\item {\ttfamily \doxylink{group__rbt__helpers_ga85f131d8278ba1dd1eff1622ca178574}{uncle()}}\+: Returns the uncle of a given node.
\item {\ttfamily \doxylink{group__rbt__helpers_ga64ec1d340ee4a76920fbd6fd1d206640}{recolor()}}\+: Handles the recolor case, fixing a double red violation.
\item {\ttfamily \doxylink{group__rbt__helpers_ga2a290397821f7fa4f417fd0589c80aef}{left\+\_\+rotate()}}\+: Rotates around a node {\ttfamily x} once to the left.
\item {\ttfamily \doxylink{group__rbt__helpers_gae3ee2dff315876e241cc8a93dddb951f}{right\+\_\+rotate()}}\+: Rotates around a node {\ttfamily x} once to the right.
\item {\ttfamily \doxylink{group__rbt__helpers_ga6351b99f0cf1652647cd859c379b459a}{restructure()}}\+: Restructures the tree with respect to a node {\ttfamily z}, fixing a double red violation.
\item {\ttfamily \doxylink{group__rbt__helpers_gac91e06a76fd1582e2dea6bfcf4576267}{fixup()}}\+: Recursively fixes up the Red-\/\+Black tree after insertion. 
\end{DoxyItemize}

{\footnotesize\ttfamily static \mbox{\hyperlink{structNode}{Node}} \texorpdfstring{$\ast$}{*} fixup (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{structNode}{Node}} \texorpdfstring{$\ast$}{*}}]{z }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [static]}}



Fixes up the Red-\/\+Black tree after insertion to maintain its properties. 

This function adjusts the colors and structure of a Red-\/\+Black tree starting from node {\ttfamily z} upwards to ensure that the tree maintains balance after an insertion. The fixup procedure includes recoloring nodes and performing rotations as necessary.


\begin{DoxyParams}{Parameters}
{\em z} & A pointer to the newly inserted node or a node that may cause a double red violation. \\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
A pointer to the root of the (valid Red-\/\+Black) tree after the adjustments.
\end{DoxyReturn}
\begin{DoxyNote}{Note}
The function recursively addresses two main cases\+:
\begin{DoxyEnumerate}
\item If {\ttfamily z\textquotesingle{}s} parent is NULL, {\ttfamily z} is the root and is colored BLACK.
\item If both {\ttfamily z} and its parent are RED, then it checks the color of {\ttfamily z\textquotesingle{}s} uncle.
\begin{DoxyItemize}
\item If the uncle is RED, we recolor.
\item If the uncle is BLACK or NULL, we restructure with respect to {\ttfamily z}. After (recursively) addressing these cases, the function ascends the tree to its root to return it. 
\end{DoxyItemize}
\end{DoxyEnumerate}
\end{DoxyNote}

{\footnotesize\ttfamily static \mbox{\hyperlink{structNode}{Node}} \texorpdfstring{$\ast$}{*} restructure (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{structNode}{Node}} \texorpdfstring{$\ast$}{*}}]{z }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [static]}}



Restructures the tree to maintain Red-\/\+Black properties after insertion. 

This function performs rotations on a given node {\ttfamily z} in a Red-\/\+Black tree to fix any violations caused by insertion. There are four cases considered for rotation\+: left-\/left, right-\/right, left-\/right, and right-\/left, each requiring specific rotations to maintain the tree\textquotesingle{}s balancing properties.


\begin{DoxyParams}{Parameters}
{\em z} & A pointer to the node that we rotate with respect to. \\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
A pointer to the node that becomes the new root of the subtree after the rotations are completed.
\end{DoxyReturn}
\begin{DoxyNote}{Note}
The function modifies the tree structure through rotations and recolors the nodes to adhere to Red-\/\+Black tree properties. The specific rotations performed depend on the position of {\ttfamily z} relative to its parent and grandparent, addressing the different cases of tree imbalance\+:
\begin{DoxyItemize}
\item Left-\/\+Left Case\+: A single right rotation is needed.
\item Right-\/\+Right Case\+: A single left rotation is needed.
\item Left-\/\+Right Case\+: A right rotation followed by a left rotation is needed.
\item Right-\/\+Left Case\+: A left rotation followed by a right rotation is needed. After the rotations, the function correctly recolors affected nodes.
\end{DoxyItemize}

The code is purposefully verbose as it serves an educational purpose. Given {\ttfamily z}, {\ttfamily p}, {\ttfamily x}, we can reduce redundancy by nesting cases as follows\+:
\end{DoxyNote}

*    if @p p is a left child:

*    else:
*      right_rotate(left_rotate())
*
*    recolor appropriately
*  else:
*    if @p p is a right child:
*      right_rotate()
*    else:
*      left_rotate(right_rotate)
*
*    recolor appropriately
*
*  return z
* \end{DoxyVerb}
